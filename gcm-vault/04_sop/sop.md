# 日常工作流 SOP

## 交付流程
1. 客戶意向確認（SurveyCake 六區塊 intake form 收資料）
2. 依 `01_services/` 對應服務檔核對範圍與定價，產出報價單
   （`05_templates/quote_template_v1_*.docx`）
3. 客戶確認報價 → 產出對應合約（`05_templates/contract_*_v1_*.docx`）
4. 合約簽署 → 導向綠界收款連結 → 收款完成（先匯後做）
5. 依服務對應交付節奏執行（顧問服務見三個月三階段，單次服務見各自交期）
6. 定稿輸出 PDF 交付客戶
7. 更新 `02_clients/` 對應客戶檔案狀態

## 金流流程（零程式碼路線）
SurveyCake 表單收資料 → 導向綠界收款連結頁 → 綠界完成收款 → SurveyCake 自訂
感謝頁轉址 → 觸發自動回信通知團隊與客戶。
SurveyCake 僅收資料，不可用於實際收款；綠界為唯一收款管道。
SurveyCake 免費版每月回覆上限 100 份，超過須升級付費方案，見
`06_facts/platform_facts.md`。

## 發案 Checklist
- [ ] 客戶意向已透過 SurveyCake 六區塊 intake form 收齊
- [ ] 服務範圍與定價已對照 `01_services/` 最新 `status: active` 版本
- [ ] 報價單已註明不二價聲明與 14 天有效期
- [ ] 合約已選對版本（履歷版／顧問版／情報書版）
- [ ] 合約款項條款為「先匯後做」，且已確認收款到位才開工
- [ ] 若涉及海外房產投報率文案，已檢核 `06_facts/real_estate_regulation.md`
      合規紅線
- [ ] 交付物為 PDF 定稿

## 異步顧問溝通規範
全文字化、異步溝通是服務設計優勢，不是限制：
- 客戶端不需配合固定會議時段
- 每次回覆需明確標示下一步行動與預期時程
- 顧問版合約明定每月 2 次會議的定義（時長、形式、是否含追加會議費用），
  避免「每日關注」被誤解為即時客服，若客戶需求超出合約定義的溝通頻率，
  依合約走追加報價流程，不得無償擴大服務範圍
